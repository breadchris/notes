Figuring it out

January 27, 2024| breadchris
<p>Someone shared <a href="https://twitter.com/karpathy/status/1751350002281300461">this</a> with me today. They said: &quot;yeah when I read this I thought of your exact thing lol&quot;. So it turns out that I am on to something. Let's take a look at how well I am meeting Andrej's ideal blogging platform:</p> <ul> <li> <p>Writing</p> <ul> <li> <p>✅ in markdown</p> </li> <li> <p>✅ with full WYSIWYG, not just split view (think: Typora)</p> <ul> <li>I am using <a href="https://tiptap.dev/">https://tiptap.dev/</a></li> </ul> </li> <li> <p>super easy to copy paste and add images</p> <ul> <li> <p>not yet, I have been itching to code this.</p> </li> <li> <p>tiptap does have an <a href="https://tiptap.dev/docs/editor/api/nodes/image">image thing</a>, which should be easy to add, but there are some more things to think through (where do the images get uploaded to?)</p> </li> </ul> </li> </ul> </li> <li> <p>Deploying</p> <ul> <li> <p>✅ renders into static pages (think: Jekyll)</p> <ul> <li>SEO optimized with <a href="https://gohugo.io/">hugo</a> and <a href="https://github.com/adityatelange/hugo-PaperMod">papermod</a></li> </ul> </li> <li> <p>✅ super simple, super minimal html with no bloat</p> <ul> <li><a href="view-source:https://breadchris.com/blog/a-sustainable-thinker-always-wins/">view-source:https://breadchris.com/blog/a-sustainable-thinker-always-wins/</a></li> </ul> </li> <li> <p>⌛ hosting at a nice url</p> <ul> <li> <p>I have my site hosted at a nice url, <a href="http://breadchris.com">breadchris.com</a>, but what is a normal user's experience?</p> <ul> <li>I want to give you: <a href="http://you.justshare.io">you.justshare.io</a></li> </ul> </li> <li> <p>What if people want to bring their own domain?</p> </li> <li> <p>What if people could buy their custom domain through the site?</p> </li> </ul> </li> </ul> </li> <li> <p>Maintaining</p> <ul> <li> <p>⌛ analytics (think: Google Analytics)</p> <ul> <li> <p>I started to write <a href="https://github.com/lunabrain-ai/lunabrain/blob/main/pkg/server/log.go#L49">the code</a>, but it isn't done.</p> </li> <li> <p>My goal is to make it look like <a href="https://simpleanalytics.com/simpleanalytics.com?from=landing">simpleanalytics</a></p> </li> </ul> </li> <li> <p>❌ comments section (think: Disqus)</p> <ul> <li> <p>TL; DR: I think there is much to explore here that I haven't thought through yet.</p> </li> <li> <p>I personally really like the clap feature of medium (their UI hides the feature very well so not many people use it, imo)</p> </li> <li> <p>I want to let people leave full length responses, but I want to give the author tools to decide to publish them or not.</p> </li> </ul> </li> </ul> </li> <li> <p>Ownership</p> <ul> <li> <p>✅ full export, access/ownership of the raw files to perpetuity should the need arise to move elsewhere.</p> <ul> <li> <p>My blog posts get written to <a href="https://raw.githubusercontent.com/breadchris/notes/master/pages/lunabrain.md">a log</a> that I include in LogSeq.</p> </li> <li> <p>The database can be sqlite, so all your content is saved into a database</p> </li> <li> <p>The code is all <a href="https://github.com/lunabrain-ai/lunabrain/tree/main">open source</a></p> </li> <li> <p>Oh, and it runs as a single static executable. Not that anyone cares about that ;) we love node_modules</p> </li> </ul> </li> </ul> </li> </ul>
